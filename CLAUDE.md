> Onboarding context for AI assistants and new team members.

## What this is

A practice/demo banking REST API built with Express + TypeScript following Domain-Driven Design,
persisting to MongoDB via Mongoose. It models bank **accounts** and money movements —
**deposit**, **withdraw**, **transfer** — with core rules like no-overdraft. It is a reference
codebase, not a production system.

## Working conventions

- **Layers:** `src/api` → `src/application` → `src/domain` → `src/infrastructure`. Business
  logic and transaction state changes live in the domain `AccountService`; application services
  orchestrate and persist.
- **Build/run:** `npm run dev` (ts-node + nodemon) or `npm run build && npm start`. MongoDB via
  `docker-compose up -d` (service `mongo`).
- **Test:** `npm test` (Jest). Only domain entities are unit-tested; there are no integration/API
  tests.
- **Lint:** `npm run lint`.
- **API responses** are wrapped in `{ success, data }` / `{ success, message, error }`.
- Domain entities carry a UUID stored in a custom `uuid` field, separate from Mongo `_id`; all
  lookups use `uuid`.

## Known caveats

- **No authentication/authorization** — every endpoint is open.
- **Transfers are not atomic** — debit and credit are separate saves with no DB transaction.
- **`Money` has no currency** — single implicit currency.
- **FAILED transactions are not persisted** — only COMPLETED ones reach the datastore.
